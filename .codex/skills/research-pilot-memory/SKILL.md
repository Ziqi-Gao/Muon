# Research Pilot Memory

Adapted from `Research-Pilot` ideas in lightweight repository form.

## Memory Files

- `docs/CONTEXT.md` or `docs/muon-context.md` for project scope and current focus.
- `sources/` or `data/source-log.csv` for source state.
- `notes/` for paper cards, math notes, and experiment notes.
- `knowledge/` for knowledge graph nodes, edges, and evidence.
- `docs/handoffs/` for session resumes.
- `docs/skill-inventory.md` for workflow provenance.

## Procedure

1. Read the latest context and handoff before acting.
2. Update structured files during work.
3. Record decisions and unknowns explicitly.
4. At session end, write a handoff.

## Rule

Memory is useful only if it changes when reality changes.

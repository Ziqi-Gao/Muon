01 问题设定2

训练到底在最小化什么？

设模型参数为一个向量。现在先不关心它来自线性层、卷积层还是 Transformer，只把所有参数摊平成：

θ∈Rd

理论上，我们希望最小化总体风险，也就是对真实数据分布的期望损失：

L(θ) = E z∼D [ ℓ(θ;z) ]

符号说明

  • L(θ)：模型在真实数据上的平均损失。
  • θ：模型参数，可以先理解成所有权重摊成的向量。
  • z：一个训练样本，通常包含输入和标签。
  • D：真实数据分布，也就是样本从哪里来。
  • ℓ(θ; z)：参数为 θ 时，在单个样本 z 上的损失。
  • Ez∼D[...]：对从 D 中抽到的样本取平均。

现实中我们只有训练集，所以用经验风险 L̂N(θ) 来估计 L(θ)：

L^ N (θ) = 1N ∑ i=1 N ℓ(θ;zi)

课堂提醒

训练时优化器真正能访问的是 L̂N，不是未知的 L。L̂N 和 L 的差距是统计误差；mini-batch 又会带来每一步的梯度噪声。

02 从 GD 到 SGD3

全量训练集梯度太贵，SGD 是它的廉价估计

如果每一步都看完整训练集，得到的是经验风险上的全量梯度下降：

θt+1 = θt - ηt ∇ L^ N (θt)

SGD 改成只看一个 mini-batch。令这一批样本的索引集合为：

Bt ⊆ {1,...,N} , |Bt|=B

mini-batch 梯度定义为：

gt = 1B ∑ i∈Bt ∇θ ℓ(θt;zi)

于是 SGD 的一行更新就是：

θt+1 = θt - ηt gt
for t = 0, 1, 2, ... sample a mini-batch B_t compute gradient estimate g_t update theta <- theta - eta_t * g_t
SGD 更新式3 / 10
03 随机近似4

SGD 的核心性质：对训练集无偏但有噪声

在常见的均匀采样假设下，mini-batch 梯度是全训练集梯度的无偏估计：

E [ gt | θt ] = ∇ L^ N (θt)

所以我们可以把它拆成真梯度加噪声：

gt = ∇ L^ N (θt) + ξt , E[ξt|θt] = 0

把这个拆分代回更新式：

θt+1 = θt - ηt ∇ L^ N (θt) - ηt ξt

确定性部分

沿着训练集经验风险的负梯度方向下降，体现优化目标的几何。

随机性部分

每一步都注入噪声，影响探索、稳定性和泛化行为。

这就是 SGD 的第二个身份：它不是单纯的下降算法，而是一个带噪声的离散动力系统。
无偏估计与梯度噪声4 / 10
04 学习率5

学习率不是旋钮，它受曲率约束

这一页把当前要优化的目标统一记作 F。训练时通常 F = L̂N；做总体风险分析时也可以令 F = L。

假设 F 是 β-smooth，也就是梯度变化不太剧烈。一个常用的不等式是：

F(θ-ηg) ≤ F(θ) - η ∇F(θ) T g + βη2 2 ||g||2

当 g 等于完整梯度 ∇F(θ) 时，下降项和二次惩罚项之间有一个竞争关系：

F(θt+1) ≤ F(θt) - η (1-βη2) ||∇F(θt)|| 2

关键解释

学习率太小，下降慢。学习率太大，二次项会压过下降项，更新可能震荡甚至发散。曲率越大，可接受的学习率越小。

SGD 更复杂，因为 g 还有噪声。粗略地说，学习率同时控制两件事：

  • 向负梯度方向走多远。
  • 把梯度噪声放大多少。
下降引理与稳定性5 / 10
05 二次型例子6

为什么病态曲率会拖慢 SGD？

看最经典的二次目标。这里假设 H 对称且正定：

F(θ) = 12 θT H θ

它的梯度是：

∇F(θ) = Hθ

全量梯度下降变成一个线性系统：

θt+1 = (I-ηH) θt

若 H 的特征值为 λi，那么每个特征方向上的误差满足：

et+1(i) = (1-ηλi) et(i)

稳定性要求大致为：

0<η< 2λmax
狭长等高线会让梯度方向来回摆动。
条件数 κ = λmax / λmin 越大，陡峭方向限制学习率，平坦方向却收敛很慢。

06 batch size7

batch size 改变的是噪声，不只是速度

若单样本梯度的协方差约为 Σ，那么 mini-batch 平均以后，梯度估计的协方差近似缩小为：

Cov ( gt | θt ) ≈ Σ B

所以 batch size B 增大时，梯度估计更稳定，但每一步的随机扰动也更小。

小 batch

  • 单步便宜。
  • 噪声较大。
  • 可能帮助跳出狭窄区域。

大 batch

  • 梯度更准。
  • 并行效率更高。
  • 学习率和 warmup 更敏感。

把噪声拆分代入更新式，可以看到噪声被学习率 η 放大：

noise in update ≈ - η ξt , Var(ηξt) ∝ η2B
这就是为什么学习率和 batch size 不能完全分开调。它们共同决定训练动力系统的温度。

07 SGD 的局限8

SGD 只做了两件事：方向和标量步长

SGD 的更新可以读成一句话：

new parameter = old parameter - (scalar) × (gradient)

它没有显式回答这些问题：

  1. 不同坐标是否应该用不同步长？
  2. 过去许多步的梯度方向是否应该累积？
  3. 矩阵参数是否应该保留矩阵结构，而不是摊平成向量？
  4. 更新方向是否应该被归一化、白化或正交化？
问题 后续优化器的回答
单步梯度太吵 Momentum 使用时间平均。
不同坐标尺度不同 RMSProp、Adam 使用坐标级自适应缩放。
矩阵方向有结构 Shampoo、SOAP、Muon 等方法显式处理矩阵几何。
所以 SGD 是所有现代优化器的零点：后面的每个方法，都可以看成对 gt 或 ηt 的改造。
SGD 作为基线8 / 10
08 通往 Momentum9

下一步：把单步梯度变成时间方向

SGD 每一步只相信当前 mini-batch 的梯度。Momentum 的出发点很朴素：如果很多步的梯度都大致指向同一方向，那这个方向更可信。

Momentum 引入速度或动量变量：

vt = μ vt-1 + gt
θt+1 = θt - ηt vt

在低噪声方向

梯度反复同向，动量会累积，走得更快。

在震荡方向

梯度来回变号，动量会互相抵消，震荡被削弱。

这一步非常重要，因为 Muon 也会先形成一个带动量的矩阵信号，再对这个矩阵信号做几何处理。

single gradient → momentum signal → structured update
Momentum 预告9 / 10
09 通往 Muon10

从向量参数到矩阵参数

神经网络里很多核心参数天然是矩阵，比如线性层权重：

W∈Rm×n

如果只照搬矩阵版 SGD，就是用当前 mini-batch 的梯度矩阵更新：

Wt+1 = Wt - ηt Gt
Gt = 1B ∑ i∈Bt ∇W ℓ(Wt;zi)

Muon 这条线的核心转弯是：先把梯度或动量看作矩阵，再对这个矩阵方向做正交化或极分解近似，得到结构化更新。

1. 梯度矩阵

当前 batch 给出 Gt。

2. 动量矩阵

累积成 Mt，降低单步噪声。

3. 几何整理

把 Mt 转成更规整的矩阵更新 Ut。

Mt = μ Mt-1 + Gt , Ut ≈ Orth(Mt)
Wt+1 = Wt - ηt Ut

这只是预告，不是完整定义。下一讲先把 Momentum 讲透，再继续到自适应方法和矩阵预条件。

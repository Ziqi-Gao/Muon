Muon 核心：动量矩阵

第五讲：Newton-Schulz、极分解与谱范数下降

Muon 的名字可以读成 MomentUm Orthogonalized by Newton-Schulz。它先形成矩阵动量，再把这个矩阵动量正交化，用谱更平坦的方向更新隐藏层权重。

本讲要回答

  • Muon 的基本更新式是什么？
  • Orth(M) = UVT 到底在做什么？
  • Newton-Schulz 为什么能近似正交化？
  • Muon 和 AdamW、Shampoo 的核心区别是什么？

本讲先不做

  • 不展开所有 2026 年变体。
  • 不把经验结论讲成定理。
  • 不讲分布式系统细节，留到 lecture06。

这组讲义会固定符号：F 表示当前训练目标，训练时通常 F = L̂N；gt 是 mini-batch 梯度，mt 是一阶动量。

01 Muon 的对象2

Muon 主要用于隐藏层的二维权重

设一个隐藏层权重为：

Wt ∈ ℝm×n

第 t 步 mini-batch 梯度为：

Gt = ∇WF(Wt)

Muon 不通常用于所有参数。标量、向量、embedding 和输出头常用 AdamW 处理；Muon 主要用于隐藏层的二维矩阵参数。

Muon 组

线性层、MLP、attention 的隐藏层矩阵。

AdamW 组

bias、norm、embedding、输出头，以及不适合矩阵化的参数。

这件事很重要：Muon 在实践中通常是混合优化器，不是全模型所有参数都用 Muon。

02 基本更新3

Muon = Momentum + Orthogonalization

先形成矩阵动量。这里用归一化动量写法，代码中也常见未归一化写法，尺度可吸收到学习率里：

Mt = μMt-1 + (1 - μ)Gt

然后对 Mt 做正交化近似：

Ut ≈ Orth(Mt)

最后更新权重：

Wt+1 = Wt - ηtUt

梯度矩阵

Gt 来自 mini-batch。

→

动量矩阵

Mt 降低时间噪声。

→

正交更新

Ut 压平奇异值。

Muon 三步3 / 12
03 Orth(M)4

如果能做 SVD，Muon 的核心一行就很清楚

令动量矩阵的奇异值分解为：

Mt = U S VT

理想正交化更新是：

Orth(Mt) = U VT

这等价于把 S 里的奇异值全部替换成 1，只保留左右奇异向量。

方向信息

U 和 V 描述矩阵更新主要连接哪些输入/输出子空间。

幅值信息

S 描述各个奇异方向的强弱，Muon 会把它压平。

Muon 不是把每个元素变成 sign，而是把矩阵的奇异值谱变平。
极分解目标4 / 12
04 为什么要压平谱？5

低秩动量会让少数方向支配学习

在许多深度网络层里，梯度或动量矩阵可能接近低秩：少数奇异方向很大，许多方向很小。

压平奇异值后，小奇异方向不会被完全淹没。直觉上，Muon 给矩阵更新更多“方向覆盖”。

这是经验动机，不要误读成所有任务上的定理。严格说法是：Orth(M) 改变了矩阵更新的谱结构。
谱压平直觉5 / 12
05 Newton-Schulz6

不用 SVD，怎样近似 Orth(M)？

Muon 使用 Newton-Schulz 类型的矩阵迭代。先归一化：

X0 = M / (||M||F + ε)

然后重复少量步骤：

Ak = XkXkT
Xk+1 = aXk + (bAk + cAk2)Xk

常见 Muon 实现使用调过的系数：

a = 3.4445,   b = -4.7750,   c = 2.0315
实现里通常会在矩阵较高时先转置，让矩阵乘法更便宜；结束后再转回来。
Newton-Schulz 迭代6 / 12
06 奇异值上的多项式7

Newton-Schulz 在奇异值上做同一个函数

若 X = U S VT，一次迭代会保持 U、V 不变，只改变奇异值：

s   →   φ(s) = as + bs3 + cs5

重复 q 次就是把 φ 反复作用在每个奇异值上：

S   →   φq(S)

设计系数的目标是让区间内的奇异值快速靠近 1，从而得到接近 U VT 的矩阵。

为什么快

只用矩阵乘法，适合 GPU；实践中常用少量步数。

为什么近似

小奇异值可能不完全到 1，因此正交化质量和步数、谱分布有关。

Muon 的高效来自这里：用低精度矩阵乘法近似本来要 SVD 才能精确得到的极分解方向。
多项式谱变换7 / 12
07 Nesterov 风格8

实践里常把 Nesterov 思想放进 Muon

Muon 可以使用普通动量，也可以使用 Nesterov 风格的动量信号。直觉是先沿已有动量看一眼，再计算新的梯度信号。

lookahead:   Wtlook = Wt - ημMt-1
Gt = ∇F(Wtlook),   Mt = μMt-1 + (1 - μ)Gt

然后仍然执行：

Ut ≈ Orth(Mt),   Wt+1 = Wt - ηUt
记住主线即可：Nesterov 改的是动量信号的形成方式，不改变“动量后正交化”这个核心结构。
Nesterov Muon8 / 12
08 维度检查9

Muon 的每个矩阵都应该形状一致

如果 Wt ∈ ℝm×n，那么：

对象形状含义
Gtm × n当前 batch 的梯度矩阵
Mtm × n动量矩阵
Utm × n正交化后的更新矩阵
Wt+1m × n更新后的权重

若 m ≥ n，理想 Ut 满足 UtTUt ≈ I；若 m ≤ n，则 UtUtT ≈ I。

这就是“半正交”：矩阵不一定方，但短边方向尽量正交。
形状与半正交9 / 12
09 与 AdamW 对比10

AdamW 缩放坐标，Muon 整理奇异值

维度AdamWMuon
主要状态mt, vtMt
缩放方式逐坐标除以 √v̂把矩阵动量正交化
几何对象坐标轴矩阵奇异向量和奇异值
适用参数几乎所有参数主要是二维隐藏层权重

所以 Muon 不是 Adam 的简单替代品。实践中常常是 Muon 和 AdamW 混用。

hidden matrices → Muon,   vectors/scalars/embeddings/head → AdamW
AdamW 是默认强基线；Muon 的价值在于对隐藏层矩阵更新做更结构化的处理。

10 与 Shampoo 对比11

Muon 更像轻量的瞬时矩阵几何

Shampoo 维护累积预条件器，Muon 只维护动量并正交化当前动量信号。

方法核心状态计算重点直觉
ShampooLt, Rt矩阵逆根长期估计左右空间曲率
MuonMtNewton-Schulz 正交化把当前动量方向谱压平
SOAP矩阵基 + Adam 状态基变换和二阶矩矩阵空间里的 Adam 风格缩放

Muon 的吸引力在于它把矩阵几何压缩成一个相对简单、矩阵乘法友好的子程序。

如果 lecture04 是“为什么 Orth 会出现”，本讲就是“如何把 Orth 做得足够快”。

11 小结12

本讲你应该带走的四句话

  1. Muon 先对二维隐藏层权重形成矩阵动量 Mt。
  2. 理想更新方向是 Orth(Mt) = U VT。
  3. Newton-Schulz 用少量矩阵乘法近似这个极分解方向。
  4. Muon 改变的是矩阵奇异值结构，不是 Adam 式逐坐标二阶矩。
Mt → Orth(Mt) → Wt+1 = Wt - ηUt

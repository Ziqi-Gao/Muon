第三讲：自适应缩放、二阶矩与 AdamW

Momentum 在时间上平均梯度，但仍然用一个全局学习率。Adam 的核心转弯是：每个坐标都维护自己的尺度估计，让噪声大或梯度大的坐标走得更谨慎。

01 从 Momentum 出发2

Momentum 只在时间上平均，没有改坐标尺度

上一讲的 Momentum 更新可以写成：

mt = μmt-1 + (1 - μ)gt,   θt+1 = θt - ηtmt

它让方向更平滑，但 ηt 仍然是一个标量。也就是说，每个坐标都被同一个学习率缩放。

时间问题

单步梯度太吵，Momentum 用过去的平均来稳定方向。

坐标问题

不同坐标的梯度尺度可能差很多，一个 η 很难同时适合所有坐标。

自适应优化器的入口就是把标量步长改成坐标级步长：

θt+1 = θt - ηtPtmt
Pt 是一个对角缩放矩阵时，就是坐标级自适应；Pt 变成矩阵或张量结构时，就通往 Shampoo、SOAP 和 Muon 的邻近地带。
从时间平均到坐标缩放2 / 10
02 对角预条件3

把每个坐标的步长分开

令第 j 个坐标的一阶动量为 mt,j。如果给每个坐标一个尺度 st,j，更新就变成：

θt+1,j = θt,j - ηt · mt,j / (st,j + ε)

这里 st,j 通常来自历史梯度平方。梯度平方大，说明这个坐标经常有大梯度或高噪声，于是步长会被压小。

Adam 的“聪明”不是知道哪个方向一定更好，而是用历史数据估计每个坐标的天然单位。
坐标级学习率3 / 10
03 AdaGrad4

AdaGrad：把所有过去的平方梯度都累加

AdaGrad 的尺度来自累计平方梯度：

vt = vt-1 + gt ⊙ gt
θt+1 = θt - η · gt / (√vt + ε)

优点

出现频繁、大梯度的坐标会自动变小步长；稀疏特征场景很有吸引力。

问题

vt 只增不减，训练后期步长可能衰减得太厉害。

AdaGrad 给了一个重要思想：学习率不必全局共享，可以从梯度历史里估计。

如果 Momentum 是“过去方向的平均”，AdaGrad 就是“过去尺度的记账”。
累计平方梯度4 / 10
04 RMSProp5

RMSProp：只记最近一段时间的平方梯度

RMSProp 把 AdaGrad 的累计和改成指数移动平均：

vt = ρvt-1 + (1 - ρ)(gt ⊙ gt)
θt+1 = θt - η · gt / (√vt + ε)

这样 vt 不再只增不减，而是主要反映最近的梯度尺度。

ρ 接近 0

尺度估计反应快，但更吵。

ρ 接近 1

尺度估计更平滑，但变化慢。

RMSProp 还没有一阶动量 mt 的完整偏差修正，但它已经把“二阶矩估计”这条线打开了。

RMSProp 可以理解为：用最近的均方梯度估计每个坐标的单位。
指数平均的二阶矩5 / 10
05 Adam6

Adam = Momentum + RMSProp + 偏差修正

Adam 同时维护一阶矩和二阶矩：

mt = β1mt-1 + (1 - β1)gt
vt = β2vt-1 + (1 - β2)(gt ⊙ gt)

由于 mt 和 vt 初始为 0，前几步会偏小，因此 Adam 做偏差修正：

m̂t = mt / (1 - β1t+1),   v̂t = vt / (1 - β2t+1)
θt+1 = θt - η · m̂t / (√v̂t + ε)
一阶矩给方向，二阶矩给坐标尺度。Adam 的核心就是这两个估计的组合。
Adam 更新式6 / 10
06 Adam 的直觉7

为什么除以 √v̂t 有用？

如果某个坐标的梯度长期很大，那么 v̂t,j 会大，这个坐标的有效步长会变小：

effective stepj ≈ η / (√v̂t,j + ε)

方向

m̂t 决定朝哪里走。

单位

√v̂t 决定每个坐标的自然尺度。

稳定

ε 防止除以太小的数。

从这个角度看，Adam 并不只是“Momentum 更复杂版”。它改变了参数空间里的度量：同样大小的 m̂，在不同坐标上会产生不同大小的参数更新。

Adam 对“坐标轴”很敏感：它假设逐坐标缩放是有意义的。这正是它和矩阵几何方法的分水岭。
坐标尺度7 / 10
07 AdamW8

AdamW：不要把 weight decay 混进自适应梯度

如果把 L2 正则直接加进梯度，Adam 会把 θ 也放进自适应分母里，导致 weight decay 和坐标缩放纠缠。

AdamW 的做法是把权重衰减从梯度更新里解耦：

θt+1 = (1 - ηλ)θt - η · m̂t / (√v̂t + ε)

Adam

梯度项和正则项可能一起被自适应分母缩放。

AdamW

先按比例收缩权重，再做自适应梯度更新。含义更清楚，也更常用。

后面讲 Muon 实践时也会遇到同一件事：weight decay 最好作为独立的参数收缩来理解。
解耦权重衰减8 / 10
08 Adam 的边界9

Adam 仍然只看坐标，不理解矩阵结构

Adam 的预条件器是对角的：

Pt = diag(1 / (√v̂t + ε))

这意味着它能区分不同坐标，却不能直接表达“行空间”和“列空间”的耦合关系。

问题Adam 的回答还缺什么
单步梯度太吵一阶矩 mt已经部分解决
坐标尺度不同二阶矩 vt已经部分解决
矩阵方向低秩或病态逐坐标缩放缺少矩阵级几何处理
这就是 lecture04 的入口：如果参数天然是矩阵，是否应该用矩阵方式缩放梯度？

09 小结10

本讲你应该带走的四句话

  1. AdaGrad 证明了“每个坐标一个学习率”是合理的。
  2. RMSProp 把历史平方梯度改成近期平均，避免步长只降不升。
  3. Adam 用一阶矩给方向，用二阶矩给坐标尺度。
  4. AdamW 把 weight decay 从自适应梯度更新里解耦出来。
Adam:   direction = m̂t,   scale = 1 / (√v̂t + ε)
